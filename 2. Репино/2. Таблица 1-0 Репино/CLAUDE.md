# CLAUDE.md — Таблица 1-0 Репино (поэтажная готовность конструктива)

**Две параллельные версии (на 2026-05-18):**
1. **Базовая** `Таблица 1-0 Репино.xlsx` — 5 колонок конструктива. Эта секция документа описывает её.
2. **Расширенная** `Таблица 1-0 расширенная Репино.xlsx` — 33 колонки по всем видам работ (зоны: «До 0» / Конструктив / «По корпусу» / «Отделка-инж.» / Общестроит). Подробности — в конце документа в разделе **«Расширенная таблица 1-0»**.

`/1-0` собирает обе одной командой.

Сводная таблица готовности **монолитных работ** по корпусам и этажам Репино в формате **1-0** (1 = готово, 0 = не готово, пусто = не требуется). Аналог «Поэтажного плана готовности помещений» из Гранита, но **только по конструктиву** (без отделки/электрики/слаботочки).

## Базовые факты

- **Объект:** Репино, СК «RBI Репино Санаторий». Корпуса **К1..К6**.
- **Источник данных:** тот же лист `МСГ` из gsheet Репино (`gdrive_file_id` = `1sb5lc_ljg19ldGA2h1jKReiRGZHgXf8XJzRNjr8DlqE`), что и в `/мсг`. Срез — **только монолитные работы** (фильтр аналогичный листу `Монолит` из `/мсг`: `X.startswith("Бетонирование") AND Y ∈ {К1..К6}`).
- **Что считается «готово» (1):** для монолита — парная Факт-строка под `План «Бетонирование …»` с заполненным AO. Для строки «Кровля» (не монолит) — парная `Факт ГПР` под `Зак. ГПР «Устройство (металлической|эксплуатируемой) кровли»` с заполненным AO.
- **Что считается «не готово» (0):** план есть (`AL = План`, `AN` заполнен), а парный факт AO пуст.
- **ГПР игнорируется как источник для монолитных строк** (Цоколь/N этаж/Мансарда). - **Выходной файл:** `Таблица 1-0 Репино.xlsx` в этой же папке (`C:\Авраменко\Claude Code Projects\МСГ\2. Репино\2. Таблица 1-0 Репино\`). В `C:\Авраменко\1. КОМПАКТ\6. Репино\` НЕ копируется (решение пользователя 2026-05-16).
- **Регулярность:** пересборка **по запросу пользователя** (`/1-0`), не еженедельно.

## Структура таблицы

**Одна большая матрица на всё Репино, корпуса подряд** (подтверждено пользователем).

### Раскладка осей

- **Строки** = `Корпус · Этаж` + отдельный блок «Лестницы/Рампы» в хвосте каждого корпуса. Корпуса идут блоками подряд.
- **Колонки = 5 конструктивов** (решение от 2026-05-16):
  1. **Плита** — фундаментная плита / плита перекрытия (для К3-К6 на «Мансарде» — плита покрытия шахт +16.700, см. ниже)
  2. **Стены**
  3. **Перекрытие**
  4. **Лестницы** — ЛК, заполняются на строках «Лестницы» в хвосте корпуса (не на каждом этаже)
  5. **Рампа** — ПР, заполняются на строках «Рампа» в хвосте корпуса (только К2)
  + столбец **Итог по строке** (`= SUM/COUNT` по 5 конструктивам — игнорирует пустые)
  + столбец **Ответственный** (по корпусу/этажу/конструктиву)

**Плита покрытия — 2 позиции для К3-К6** (решение от 2026-05-16). У типовых корпусов плита покрытия со ступенькой на отм. **+13.400** (основная) и **+16.700** (шахты). В матрице это:
- Строка «4 этаж» × колонка «Перекрытие» = плита покрытия основная (+13.400)
- Строка «Мансарда» × колонка «Плита» = плита покрытия шахт (+16.700) + строка «Мансарда» × «Стены» = стены шахт

**Мансарду включаем как отдельный этаж** (решение от 2026-05-16) — несмотря на то, что у К1 над 2 этажом монолита нет (деревянная стропильная кровля), у К3-К6 на мансарде монолит частичный (только шахты). У К2 мансарды нет (3 этажа + плита покрытия → сразу кровельные работы).

- **К2** — КЖ2 (`Прим 441-РД-КЖ2`, 74 стр.): фунд. −3.250 → ПП0-1 −0.100 → ПП1-1 +3.200 → ПП2-1 +6.150 → ПП3-1 (плита покрытия). + ЛК1, ЛК2, ЛК3 + **рампа ПР0-1** (подвал→1) и **ПР1-1** (1→2).
- **К3-К6** (типовые) — КЖ3.0 (`Прим 441-РД-КЖ3.0`, 27 стр., ниже 0.000) + КЖ3.1 (`Прим 441-РД-КЖ3.1`, 67 стр., выше 0.000): фунд. −3.000 → ПП0 −0.200 → ПП1 +3.200 → ПП2 +6.500 → ПП3 +9.800 → плита покрытия ступенчатая +13.400/+16.700. + ЛК1 + монолитные лестницы подвала.

У К1 «2 этаж» — последний жилой; пространство над ним под скатной кровлей деревянное.

**Маркировка перекрытий разная**: у К1/К2 — `ПП<N>-1` (с суффиксом), у К3-К6 — `ПП<N>` (без). Учитывать в `extract_filter.match_rules` для этапа 2.

### Формулы подсчёта (1/0 → проценты)

- **Итог по строке (`Готовность помещения` → у нас «Готовность этажа»):** `=IF(COUNT(C{r}:E{r})=0,"",SUM(C{r}:E{r})/COUNT(C{r}:E{r}))`. Формат — процент.
- **Итог по столбцу (заголовок этажа корпуса):** `=IF(COUNT(C{r1}:C{r2})=0,"",SUM(C{r1}:C{r2})/COUNT(C{r1}:C{r2}))` по диапазону строк корпуса.
- **Итог по корпусу:** строка-разделитель внизу блока корпуса со сводным процентом.
- **Итог по объекту:** сводная строка/блок в начале или конце таблицы — общая готовность Репино по конструктиву.

Пусто (`""`) при `COUNT=0` — чтобы не показывать `#DIV/0!` и не учитывать в дальнейших агрегатах.

## Расширенная таблица 1-0 (2026-05-18)

`Таблица 1-0 расширенная Репино.xlsx` — 33 колонки × 90 строк, 5 зон по горизонтали.

### Зоны шапки

| Зона | Колонки | Применимость | Источник |
|---|---|---|---|
| Корпус/Этаж (fix) | A, B | label | — |
| **«До 0»** | C..J (8) | строка «Цоколь» | Зак.ГПР + парный Факт ГПР |
| **Конструктив** | K..O (5) | этажи (Плита/Стены/Перекр.) + Лест./Рампа | как в базовой |
| **По корпусу** | P..W (8) | строка «По корпусу в целом» | Зак.ГПР + парный Факт ГПР |
| **Отделка/инж.** | X..AF (9) | этажи 1..N + Мансарда | **дефолт 0** (источника по этажам в МСГ нет, ручной ввод) |
| Итог | AG | формула % | — |

### Колонки

- Зона А (C..J): Подгот./Котлован/Шпунт/БК/ГИ гор./ГИ верт./Обр.засыпка/Утепл. ниже 0
- Конструктив (K..O): Плита/Стены/Перекр./Лестн./Рампа
- Зона В (P..W): Кровля/Окна Al/Витражи/Фасад/Лифты/МК/Двери/Сборн.каркас
- Зона Б2 (X..AF): Перегор./Стяжка/Отопл./Водосн./Водоотв./Вентил./ЭС/Слаботочка/Отделка

### Особые applies

- **Шпунт** активен только для К1, К2, К3 (К4-К6 серый N/A)
- **Окна Al** активны для К1, К3-К6 (К2 серый — там витражи)
- **Сборн.каркас** активен только для К3-К6 (К1-К2 серый)

### Структура строк блока корпуса

1. Заголовок «Корпус Кi» (merged)
2. **«По корпусу в целом»** — спец.строка, активна только Зона В
3. Этажи (Цоколь, 1, ..., Мансарда, Кровля)
4. Лестницы (свои строки)
5. Рампы (только К2)
6. Итог по корпусу (формулы)

### Хвост (R83..R90)

- R83 — заголовок «Общие по объекту»
- R84..R87 — ИТП / Благоустройство / Наружные сети / Ввод (data-колонки серые, активна только AG)
- R88 — **ИТОГО общестроительные**: `=IFERROR(AVERAGE(AG84:AG87),"")`
- R90 — **ИТОГО по объекту Репино**: `=IFERROR(AVERAGE(<6 итогов корпусов>, AG88),"")`

### Логика «1» для разных зон

| Вид работ | Логика |
|---|---|
| Подготовка | AND по 4 подвидам (Y=null → одно значение на все 6 корпусов) |
| Котлован, Шпунт | один Зак.ГПР, Y=К1..К6 |
| БК | один Зак.ГПР, Y='БК' → во все 6 |
| ГИ ниже 0 (4 вида) | AND по корпусам К1..К6 + К1-К6 |
| Кровля | AND по 2 подвидам (металл./эксплуатируемая), Y=К1..К6 |
| Окна/Витражи/Фасад/Лифты/МК/Двери/Сборн.каркас | один Зак.ГПР на корпус |
| ИТП/Благоустройство/Нар.сети/Ввод | AND по ВСЕМ Зак.ГПР раздела |

«1» = парный Факт ГПР под этим Зак.ГПР с заполненным AO.

### Легенда

`1 = готово · 0 = не готово · пусто = не требуется`. Серая заливка = ячейка не применима (например, Шпунт для К5).

### Дубль «Кровля»

В строке «Кровля» × M (Перекр.) **и** в «По корпусу в целом» × P — одно и то же значение в двух местах. Это сознательное дублирование: первое участвует в этажном итоге корпуса, второе — в Зоне «По корпусу».
