## Что такое МСГ

**МСГ — Месячно-Суточный График** — рабочий инструмент еженедельного контроля стройки. По сути это сводная Google-таблица, в которой:

- зафиксирован полный перечень работ по объекту (ГПР);
- по каждой работе ведутся **план/факт** на всём жизненном цикле (от РД до СМР);
- ежедневно/еженедельно проставляется фактический процент выполнения и фактические даты;
- автоматически считается отставание/опережение в днях и в объёмах на отчётную дату.

МСГ ведётся **онлайн в Google Sheets** и заполняется **удалённо ответственными исполнителями** (кураторами разделов, ОКС, тендерным отделом, финансами, подрядчиками). Каждый ответственный отвечает за свой блок строк в своих столбцах.

**Источник (мастер-таблица):**
[https://docs.google.com/spreadsheets/d/1sb5lc_ljg19ldGA2h1jKReiRGZHgXf8XJzRNjr8DlqE/edit?gid=1576230551#gid=1576230551](https://docs.google.com/spreadsheets/d/1sb5lc_ljg19ldGA2h1jKReiRGZHgXf8XJzRNjr8DlqE/edit?gid=1576230551#gid=1576230551)

## Структура файла МСГ

В Google-таблице **2 листа**:

Размер: **≈ 4 240 строк × ≈ 990 столбцов** (большая часть столбцов — ежедневная сетка дат от авг.-2025 до янв.-2028).

**Логические блоки шапки (строки 1–4):**

| Блок | Колонки | Назначение |
|---|---|---|
| Календарь / праздники | A–B | Контрольные даты строительства и календарь праздников 2025–2027 |
| Атрибуты раздела | C–F | Подрядчик, Раздел РД, Подраздел РД, Удельный вес |
| Расчёты по периоду | I–M | Отклонение к началу периода, План/Факт в периоде, Прогноз % завершения, Факт-остаток на сегодня, Проверка плана |
| Идентификация работы | O–Y | Фильтр по месяцам, Ответственный, ТЭГ, ИД, Готовность предшественника, Наименование работы, Здание (К1–К6 / БК / ИК / общее) |
| Объёмы | AA–AD | Отклонение/опережение по объёму, единица измерения, проектный объём |
| План/Факт на сегодня | AF–AJ | План/факт с начала стройки, % завершения, отклонение в днях по началу и по окончанию |
| Жизненный цикл (ключевая колонка) | AL (= c38 «План / Факт») | **Маркер этапа**: Зак. ГПР / Раб. ГПР / Факт ГПР / План РД / Факт РД / Пакет / Тендер / Договор / Финансирование / Мобилизация / План работ / Факт работ |
| Сроки этапа | AN, AO | План/Факт начало, План/Факт окончание |
| Месячная сетка | AQ | План/Факт по месяцам |
| Суточная сетка | AR → … | По одной колонке на каждый день стройки (~ 700 столбцов), где проставляется ежедневный план/факт |

**Главный принцип** — каждая работа в ГПР развёрнута в **14 строк жизненного цикла**:

1. **Зак. ГПР** — заказная (директивная) дата
2. **Раб. ГПР** — рабочая дата
3. **Факт ГПР** — фактический срок по графику
4. **План РД** — выпуск рабочей документации (план)
5. **Факт РД** — выпуск РД (факт)
6. **Пакет** — пакет документов на тендер
7. **Тендер** — проведение тендера
8. **Договор** — заключение договора с подрядчиком
9. **Финансирование** — выдача аванса
10. **Мобилизация** — мобилизация подрядчика и МТР
11. **План работ** — план СМР
12. **Факт работ** — факт СМР
13–14. служебные строки (С38, итоги)

Для аналитики по проценту выполнения СМР используются строки `План` и `Факт` (тип C38).

## Как ведётся МСГ

1. **Удалённое заполнение онлайн.** Каждый ответственный заходит в Google-таблицу со своим аккаунтом и обновляет свой блок: проставляет фактические даты, % выполнения, фактические объёмы по своему разделу/этапу.
2. **Регулярность.** Минимум — еженедельно перед пятницей (на отчётную дату). По активным работам — фактически ежедневно.
3. **Источник истины — таблица в Google.** Локальные `.xlsx`-копии используются только как срез для отчётности, в них руками ничего не правится.
4. **Шапка и календарная сетка не меняются** — обновляются только ячейки факта в строках работ и в суточной/месячной сетке.

## Еженедельная выгрузка отчёта (ПЯТНИЦА, утро)

**Каждую пятницу утром** из мастер-таблицы готовится отчёт «**МСГ. КРИТИЧЕСКИЕ ОТСТАВАНИЯ**» (PPTX + PDF) для рассылки в ленту Битрикс.

**Сборка автоматическая, через slash-команду в Claude Code:**

```
/мсг2 2.репино
```

(точное имя ключа — папка `projects\<key>\config.json`; для этого объекта планируется ключ, согласованный с раскладкой в `C:\Авраменко\Claude Code Projects\МСГ\projects\`)

Конвейер `/мсг2` делает:

1. читает `gdrive_file_id` из `projects\<key>\config.json`;
2. выгружает свежий CSV с листа «МСГ, ГПР» из Google Sheets;
3. фильтрует позиции с отрицательным отклонением по объёму (`obj_otkl < 0`) и без полностью заполненного факта (нет одновременно «Факт начало» + «Факт окончание»);
4. фильтр применяется ко всем этапам ЖЦ (РД, Пакет, Тендер, Договор, Финансирование, Мобилизация, СМР);
5. собирает PPTX по шаблону `template\repino_template.pptx` с обложкой `cover.png`, нумерацией слайдов и заголовком формата:

   `МСГ. КРИТИЧЕСКИЕ ОТСТАВАНИЯ | НЕДЕЛЯ XX (ДД.ММ–ДД.ММ.ГГГГ)`

6. конвертирует PPTX → PDF через PowerPoint COM;
7. кладёт оба файла в `output_dir`, указанный в конфиге.

**Папка для готовых отчётов по этому объекту:**

```
C:\Авраменко\1. КОМПАКТ\Отчеты\Еженедельные отчеты по пятницам\Репино
```

Именно этот путь нужно прописать в `output_dir` в `config.json` (в JSON — со слешами `/` или экранированными `\\`).

**Что нужно для запуска по этому проекту:**

- создать `C:\Авраменко\Claude Code Projects\МСГ\projects\2.репино\config.json` (по образцу `projects\бугры-3\config.json`),
  с полями: `name`, `project_full = "Санаторно-курортный комплекс «RBI Репино Санаторий»"`, `project_short`, `project_addr`, `report_basename`, `gdrive_file_id = "1sb5lc_ljg19ldGA2h1jKReiRGZHgXf8XJzRNjr8DlqE"`, `cover_image`, `template_pptx`, `output_dir = "C:/Авраменко/1. КОМПАКТ/Отчеты/Еженедельные отчеты по пятницам/Репино"`;
- положить рядом `cover.png` (обложка для презентации);
- убедиться, что у пользователя, под которым работает MCP Google Drive, есть доступ к таблице.

После этого еженедельная выгрузка по пятницам запускается одной командой `/мсг2 2.репино`.
