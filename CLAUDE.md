## Конвейер `/мсг <project-key> [month]` (xlsx к совещанию)

**Что делает:**
1. `gsheet_download_xlsx.py` качает gsheet целиком через прямой URL `https://docs.google.com/spreadsheets/d/<id>/export?format=xlsx` (~12 МБ, полное форматирование, conditional formatting, формулы).
2. `build_meeting_report.py` (Excel COM, win32com) удаляет лишние листы (РС), переименовывает источник в `МСГ`, клонирует его в `ГПР` и пять стадийных листов, ставит на каждом свой фильтр и view tweaks.

**На выходе:** `<meeting_out_dir>\МСГ_<basename> отчет к совещанию от <DD.MM.YYYY>.xlsx`. Пути берутся из `cfg["meeting_out_dir"]` (CLI `--out-dir` → config → fallback).

### Структура исходного gsheet — колонки МСГ-листа

A пусто · B праздники 25-26-27 · C-F подрядчик/раздел/подраздел/уд.вес · G-H формулы дней · I-M аналитика · **O — «Фильтр по месяцам»** (рабочая колонка, в `O2` — реальная работа, **не селектор**) · Q ответственный · R ТЭГ · T ИД · V готовность предшественника · X наименование · Y здание · AA объём откл. · AC ед. · AD проектный объём · AF план/факт с начала · AG % завершения · AI/AJ дни отклонения · **AL — стадия** · AN/AO план/факт начало/окончание · AQ план/факт месяц · **AR..AKY (~944 кол.) — календарь по дням** от 1 авг 2025 до 2028.

Реальные значения `AL`: «Зак. ГПР» / «Раб. ГПР» / «Факт ГПР» / «План» / «Факт» / «План|Факт РД|П|Т|Д|Ф|М». **Нет** значения «План ГПР».

### Листы и фильтры

- **МСГ** — AutoFilter `M1:AO`, Field=3 (O) = `<month>`. View: outline до месячных разделителей, show `X:AO` и колонки отчётного месяца в Гант, hide `A:W`, оставить `V`. `AI2:AJ` пересоздать conditional formatting (только `Font.Color`: `<0` → красный 255, `>0` → тёмно-зелёный 32768, без заливки).
- **ГПР** — AutoFilter `M1:AO`, Field=26 (AL) ∈ `(Зак.ГПР, Раб.ГПР, Факт ГПР, =)`. Hide `A:W`, `Z:AK`, `AQ`; outline до уровня 1; **AG не скрывать** (после hide-блока `gpr.Columns("AG").Hidden = False`); AN:AO явно показать.
- **РД / Пакет / Тендер / Договор / Мобилизация** — все 5 идут через одну функцию `build_stage_sheet(ws, label, plan_tag, fact_tag)` (см. ниже).
- **Монолит** — отдельный лист, не клон. Шаблон 13 строк (фундамент + 6 уровней стен + 6 уровней перекрытий), 6 корпусов × {План, Факт}. Только для Репино (см. `projects\2.репино\CLAUDE.md`). Для остальных в config — `"include_monolit": false`.
- **Предписания СК** и **Документы подрядчики** — пока НЕ создаются. Ждут вебхук Битрикс-24, оттуда будут тянуться программно.

**Итого сейчас 8 листов** (МСГ + ГПР + 5 стадий + Монолит для Репино). Целевых 10 — добавятся два «битриксовых».

### Правило `build_stage_sheet` (group-aware с подработами, 2026-05-13)

1. AutoFilter по `AL ∈ {Зак.ГПР, Раб.ГПР, plan_tag, fact_tag, blank}`.
2. **Work-блок** = строки от `AL='Зак. ГПР'` до следующей такой же.
3. **Подработа** = строки от X-заполненной до следующей X-заполненной внутри блока. **Исключение:** строки с `AL ∈ {Раб.ГПР, Факт ГПР}` НЕ начинают новую подработу, даже если их X заполнен (иногда там имя подрядчика — `СМР - ООО "ШефСтрой"`). Без этого исключения шапка обрывается на одной строке `Зак.ГПР` (баг Бугры-3 2026-05-13).
4. Блок qualifying ↔ есть строка с `AL ∈ tags AND AA<0`.
5. В qualifying-блоке показываем шапку + только подработы с AA<0. Остальные подработы того же блока скрываем (важно для договоров К1-К2 / К3-К4 / К5-К6 — видим только qualifying-кусок).
6. Section header (AL пуст, X заполнен) виден, если в его секции есть qualifying-блок.
7. View: outline collapsed; `X:AG` + `V` + `AN:AO` видны; `A:W` (кроме V), `AH:AK`, `AQ` скрыты.
8. **Tab-color:** если `blocks_with_neg` пуст (видна только R1) → `ws.Tab.Color = TAB_COLOR_OK` (BGR 13561798, Excel «Good»). Подкраска сбрасывается между запусками — клонируем МСГ без tab-цвета.

**Правило «нет минусов → одна строка»** (РД и аналогично): если ни в одном блоке нет AA<0 в tag-строках, на листе остаётся только R1 (шапка). Никаких placeholder'ов «нет данных» не отрисовывать. Реализовано автоматически: `hide_rows` пропускает только `i==0`.

Бэкап до рефакторинга — `build_meeting_report_v_before_refactor.py` (990 строк, 5 копий логики). Текущая версия — 579 строк, контрольные счётчики совпали.

### Технические тонкости `/мсг`

- **Префикс `_xlfn.`** Google Sheets экспортирует `IFS/SWITCH/MAXIFS/...` без префикса, русский Excel ломается (`#NAME?`, в формуле «IFS» вместо «ЕСЛИМН»). В скрипте есть `patch_modern_funcs(xlsx)` — открывает zip, regex по `xl/worksheets/sheet*.xml` дописывает `_xlfn.`. Вызывается ДО открытия в Excel COM, ПОСЛЕ копирования в ASCII temp. На Репино правит ~461 формулу.
- **Patch применяется ОДИН раз — клоны наследуют автоматически.** `Worksheets("МСГ").Copy(...)` копирует уже исправленные формулы.
- **`xl.CalculateFull()`** обязателен ОДИН раз перед всеми стадийными листами — иначе формульные `AA` не пересчитаны.
- **`ShowLevels(1)` ПЕРЕД ручным скрытием колонок и ПЕРЕД unhide `AN:AO`.** Иначе outline collapse перезатирает ручные `hidden`.
- **openpyxl `column_dimensions[L].hidden` может врать** — для одиночной буквы создаёт default-запись, не пересекая range-записи. Для верификации читать `<col min=... max=... hidden=.../>` напрямую из `xl/worksheets/sheet*.xml`.
- **НЕ писать в `O2`** — это рабочая ячейка «ПОДГОТОВИТЕЛЬНЫЕ РАБОТЫ - Горошков», не селектор месяца. Прошлая версия скрипта писала туда `<month>` и портила фильтр. Текущий не трогает.

## Конвейер `/мсг2 <project-key>` (PPTX+PDF в Битрикс)

**Что делает:** `extract_items.py` (CSV → items.json) → `build_report.py` (PPTX из items.json + config + шаблон) → `convert_pdf.py` (PPTX→PDF через PowerPoint COM). На выходе `МСГ критические отставания <REPORT_BASENAME> неделя <WEEK>.pdf` (без префикса `2. `).

### Технические тонкости `/мсг2`

- **PowerPoint COM SaveAs не работает с кириллическими путями** — конвертация через ASCII temp, потом copy в `C:\Авраменко\...`.
- **Pptx после клонирования слайдов** нужно прогнать через `Presentation.save()` (python-pptx) для нормализации XML — иначе COM SaveAs падает с `E_FAIL`.
- **Удаление слайдов отложено** до конца (после всех `clone_slide`), иначе python-pptx переиспользует освобождённые `partname` и в zip получаются дубли `slideN.xml`.
- **Excel COM скриншот для ГПР-слайда:** `Range.CopyPicture(Appearance=1 xlScreen, Format=2 xlBitmap)` → `PIL.ImageGrab.grabclipboard()` → PNG. Перед `CopyPicture` форсить `ActiveWindow.View = 1` (xlNormalView) — иначе появляется водяной знак «Страница N» из Page Break Preview.
- **Колонка «Здан.»** наследуется всеми 16 строками жизненного цикла. В `extract_items.py` `prev_block` сбрасывается ТОЛЬКО на `Зак. ГПР` — иначе на РД/Пакет/Тендер/... колонка пустая.
- **CSV-числа с NBSP** (`\xa0`) как разделитель тысяч — парсер чисел обрабатывает.
- **Слайды нумеруются `X/N`** после пагинации — поиск шейпов с текстом по regex `^\d+/\d+$`.
- **Авто-нормализация pct (21.05.2026).** В gsheet «% завершения» (колонка AG) у разных объектов в разных форматах: Репино/Марьино — проценты (0..100), Бугры-3 — доли (0..1). Сразу после `ITEMS = json.load(f)` пробегаемся по `pct`; если <5% значений > 1.5 → это формат-доли, умножаем все на 100. Без этого Жизненный цикл МСГ показывает «0.6/0.4/...» вместо «60/40/...». Логика срабатывает только когда нужно — для Репино/Марьино no-op.
- **`show_contractor=False` для РД (21.05.2026).** `build_paged_table`/`paginate_by_section`/`build_data_table` принимают параметр. При False — `_section_key` игнорирует `contractor`, и section-row отрисовывается без em-dash + имени подрядчика. По умолчанию True; в коде явно False только в `build_paged_table(prs, slide4, RD_LAG, ...)`. На стадии РД подрядчик ещё не релевантен (это будущий генподрядчик СМР).
- **Подрядчик из МСГ (20.05.2026).** На строке `Раб. ГПР` сразу после `Зак. ГПР` в колонке X лежит имя подрядчика, если он назначен (например `ООО "АВТОДОРКОМПЛЕКС"`). Если X равен названию работ — подрядчика нет. `extract_items.py` собирает `current_contractor` по этому правилу; `lagging_by_stage`/`lagging_smr` пробрасывают в out; `_section_key(it)` = `(section, contractor)`. На section-row выводится `{раздел}  —  {подрядчик}` (раздел bold navy, подрядчик обычным, тот же navy).
